# Session 2026-05-18 — Stage 5d (React UI) + Per-segment style

## TL;DR

Built the full React UI panel for the marking tool, ironed out the local-mod
manifest pipeline (CS2 doesn't quite hand it to you out of the box), shipped
two hover-bridges and a per-segment popover overlay, fixed G87 brightness via
multi-pass spawning, added Double Solid, and migrated style ownership from
line → segment so each piece of a line can render differently.

Mod is functionally complete for the v2 killer-feature set. UI polish is
next session.

## What landed

### Build pipeline (the long fight)

- React UI lives in `src/TownRoadLaneUI/` — webpack + TS + SCSS bootstrapped
  from the `create-csii-ui-mod` template + SystemTimeMod cross-reference.
- Initial mount used the SystemTimeMod `ExecuteScript`-into-body hack. Worked,
  but triggered a `NullReferenceException` in `UIModuleAsset.PostCreate` every
  load — game synthesised an asset for our `.mjs` and called `AddTags(null)`.
- Switched to the **official `moduleRegistry.append(...)` pipeline** with
  components in `GameTopLeft` (toolbar button) + `GameTopRight` (panel) slots.
  Now matches how every other CS2 UI mod registers.
- Build wiring: webpack outputs to local `dist/`. A new `BuildFrontend`
  MSBuild target inside `TownRoadLane.csproj` runs `npm run build` and copies
  `dist/*.mjs/.css` + `mod.json` into `$(OutDir)`. Then the stock `Mod.targets`
  mirrors the whole `OutDir` into `Mods/TownRoadLane/`. Both ends survive
  rebuilds (used to wipe each other before).
- Author switched to `mxerf` everywhere.

NullRef in `AssetData.AddTags(null)` is **still in the log** — looks like
a Colossal bug that fires regardless of manifest contents, but the UI
module registers fine right after (`Registered UI Module ...` in Modding.log)
so it's purely cosmetic. Left for pre-release polish.

### Bridges (C# ↔ React)

Three connection points between the panel and the tool:

1. **State publishing (`GetToolState`)** — JSON snapshot each tick:
   `{ isActive, selectedNodeIndex, currentStyle, lastClickedLine,
   lastClickedTick, lines: [{ lineIndex, style, segments: [{ ..., screenX,
   screenY }] }] }`. React `useValue` auto-resyncs.
2. **Commands** — `TriggerBinding` callbacks: `ToggleSegment`, `SetLineStyle`,
   `SetSegmentStyle`, `DeleteLine`, `ActivateTool`, `SetHoveredLine`.
3. **Hover bridges (both directions)**:
   - **UI → game**: hover a line row in the panel → cmdSetHoveredLine →
     overlay draws that line in yellow/thick.
   - **Game → UI**: cursor over a line in the world (NodeSelected state,
     not on a dot) → `HitTestLines()` → `_hoveredLineInGame` → same overlay
     highlight. Sampling-based distance-to-Bezier, 12 samples per curve.
   - **Click in world**: click a line in NodeSelected → `_lastClickedLine`
     bumped via tick counter → React `useEffect` auto-expands that accordion.

### Per-segment popover

Each segment of the **currently-expanded** line gets a floating popover above
its midpoint in world space.

- C# computes `MathUtils.Position(fullBezier, (tStart+tEnd)/2)`, runs it
  through `Camera.main.WorldToScreenPoint`, publishes `screenX/screenY`
  (with Unity→CSS Y-flip) in the segment JSON.
- React renders via `createPortal` to `document.body` — needed because
  `GameTopRight` likely creates a CSS containing block that pins
  `position: fixed` to the slot, not the viewport.
- Popover buttons: toggle visibility, cycle per-segment style. Both wire
  back through existing commands.

### Per-segment style (data model bump)

Style moved from `MarkingLine` → `MarkingSegment`:

- `MarkingSegment.style` added. Schema v2; old saves deserialize with
  `style = 0` (Solid).
- `MarkingLine.style` stays as the **default** for newly-created segments.
- `MarkingTopologySystem` inherits style from the parent segment on split
  (`LookupInheritedStyle`, same shape as `LookupInheritedVisibility`).
  Falls back to `MarkingLine.style` for genuinely-new segments.
- `MarkingSegmentEmissionSystem` reads `seg.style` instead of
  `lines[seg.lineIndex].style`.
- `SetLineStyle` command — sweeps every existing segment of the line to the
  new style (acts as a "reset all" gesture).
- New `SetSegmentStyle(line, seg, style)` — overrides one piece.

### Stage 5c rendering wins

- **Double Solid** added — vanilla mesh `White Double Solid Line Mesh`
  cloned onto `Car Bay Line` archetype. One spawn, two visible parallel
  lines (mesh has the double pattern baked in).
- **G87 brightness** fixed via `DrawPasses()` extension on `MarkingStyle`.
  G87 styles return 3, vanilla styles return 1. Emission loops the spawn
  per pass; `TRLSegmentLink` gained `passIndex`, PathNode slot layout
  expanded to `32768 + lineIndex*512 + segmentIndex*16 + passIndex*4`.
  Three stacked semi-transparent copies match parking-line G87 brightness.
- G87 source prefab moved from `Highway/Lane Line` to `Car Bay Line` (same
  as parking) for consistency.
- `hostOnCityLanes=false` for every non-edge-line clone — clears all three
  host arrays so vanilla SecondaryLaneSystem doesn't auto-host them.
  Earlier rev had Dashed/G87 leaking into vanilla markings.

### Overlay polish

- Drag preview: thin (`0.10m`), white, semi-transparent — like a chalk
  guide line. Old version was a thick green Bezier that obscured the road
  paint underneath.
- Endpoint dots: `1.10m` (was 1.60), thin outline, style-tinted fill
  (orange = Solid, blue = Dashed).
- Hover dot bumps to `1.45m` with thicker outline + brighter alpha.
- Source dot bright white, fully filled.
- Hidden segment ghost made softer (0.30 alpha vs 0.35).

### Defensive UI

- `PanelErrorBoundary` wraps the whole panel — JS exceptions inside our
  React tree don't tear the game HUD down. Caught errors render a small
  red "Panel error" placeholder with a Retry button.
- `useToolState` normalises every field of the parsed JSON — caller can
  trust `state.lines` is an array, every line has `segments` array, etc.
  Prevented `Cannot read properties of undefined (reading 'length')`
  cascades from intermediate ticks.
- Native `<select>` element triggered a cohtml JS runtime exception on
  click — reverted to button row. To investigate in polish pass.

## Earned invariants

- **Manifest in local mods**: ship a `mod.json` next to `.mjs` for the
  game to register the UI module. Manifest must be minimal; extra fields
  like `displayName`/`description`/`isCodeMod` don't help and the NullRef
  fires regardless. (Hot take: just accept the noisy log line.)
- **`moduleRegistry.append`**, not `ExecuteScript`. The body-mount hack
  from SystemTimeMod technically works but produces an asset-registration
  error every load.
- **Portal screen-positioned popovers to `document.body`**. Don't trust
  `position: fixed` inside a CS2 UI slot — slots have transforms.
- **G87 styles need multi-pass spawning** to match vanilla brightness.
  Single decal is ~30% dimmer than two overlapping copies; parking
  inadvertently gets two via its `m_LeftLanes + m_RightLanes` hosting.
- **Style ownership lives on segment, not line** — per-piece overrides
  are the whole point of segmentation; line-level style is just a
  default for fresh segments.
- **`hostOnCityLanes=false` for archetype-only clones** — anything not
  meant to render through the vanilla SecondaryLane pass must clear all
  three host arrays after `DuplicatePrefab`. See
  `memory/project-prefab-clone-hosting.md`.
